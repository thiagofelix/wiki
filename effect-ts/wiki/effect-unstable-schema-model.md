# Model (unstable)

Base class for domain model schemas that support multiple variants for database and JSON APIs. Built on top of `VariantSchema` with preset variants `select`, `insert`, `update`, `json`, `jsonCreate`, and `jsonUpdate`, letting one declaration serve reads, writes, and wire formats.

## Key Exports
- `Class` — base class constructor producing a Schema with `.insert`, `.update`, `.json`, `.jsonCreate`, `.jsonUpdate` variants
- `Struct` — non-class variant struct constructor
- `Union` — union of variant schemas
- `Field` — define a field with per-variant schemas
- `FieldOnly` / `FieldExcept` — restrict a field to specific variants
- `fieldEvolve` — transform per-variant schemas of a field
- `Generated` — column generated by the database (select/update/json only)
- `GeneratedByApp` — column generated by the application (all DB + json)
- `Override` — brand marking a value as an override
- `extract` — extract a single variant schema from a struct
- `fields` — accessor for the underlying variant fields
- `VariantsDatabase` / `VariantsJson` — type aliases for variant groupings
- `Any` — top type for Model schemas
